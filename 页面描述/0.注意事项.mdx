# 注意事项

编写 JSON 过程中的注意事项。

## 命名

- 由于一些历史原因，使用 `type : [component]`指定组件时，组件首字母需小写，如使用`InputNumber`则描述为`"type" : "inputNumber"`。这个问题将会在下一个大版本被解决。

- columns 中的对象 key 和 label 需保持一致，这个也是历史原因，会在下一个大版本修复。

## 报错

由于该项目诞生之初主要是为了提高团队内部生产力，在约定俗成的写法描述下，并没有做细粒度的抛错处理，所以开发者可能会在编写 JSON 调试界面的过程当中遇到界面为空的情况，这种情况一般会有如下几个原因：

- JSON 描述不规范，如写错组件名称，写错某些 props key 值等。

- 数据解析错误，解析出来的数据格式和组件需要的数据格式不匹配。

## 解决方案

### 关于报错

我们会在之后的迭代过程中为组件添加容错机制，为开发者提供更好的调试体验，同时在下一个大版本中，这一部分将会被重构，重构之后将会在架构层面添加更多的容错机制。
