# Chart & Kanban 图表和看板

图表和看板，用来对结构化数据进行可视化，以呈现更为清晰且易于理解的数据指标。

## 使用场景

- 作为图表或看板页面

- 供 Table 上的 ChartModal 调用，以便快捷浏览指定数据图表

## 示例

<Notice type='warning'>
	注意：Chart组件既可作为页面使用，也可作为单独的组件使用。大屏和看板的区别是看板没有筛选工具栏，且右上角的按钮能链接到大屏。
</Notice>

## Props

```ts
type ChartComponent = 'numberCard' | 'table' | 'line' | 'bar' | 'lineBar' | 'map' | 'pie'

interface IProps {
	// 页面标题
	label: string
	// 版本
	version?: string
	// 页面描述
	description?: string
	// 参考 Yao 关于 Node 的定义
	nodes: Array<Node>
	// 参考 Yao 关于 Api 的定义
	apis: Api
	// 参考 Yao 关于 Output 的定义
	output: Output
	// 参考 Table & Form 章节的Columns
	filters?: Columns
	// 页面描述
	page: {
		// 主键
		primary: string
		// 页面为看板时有效
		option?: {
			// 数据刷新间隔，单位毫秒
			request_interval: number
			// 大屏链接
			screen: string
		}
		layout: {
			// 筛选项
			filters?: Array<{
				name: string
				width?: number
			}>
			// 图表
			charts: Array<Chart>
		}
	}
}

interface Chart {
	// 图表名称 & 数据 Key
	name: string
	// 宽度，24栅格
	width: number
	// 组件名称，charts字段有值时可不指定
	type?: ChartComponent
	// 组件属性，charts字段有值时可不指定
	props?: { [key: string]: any }
	// 是否在图表底部显示 table
	table?: boolean
	// table相关属性
	table_props?: {
		// 高度，超过滚动
		height: number
		// 列信息
		columns: Array<{
			title: string
			dataIndex: string
		}>
	}
	charts?: Chart | Array<Chart>
}
```
